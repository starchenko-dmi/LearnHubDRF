# Образовательная платформа на Django REST Framework

REST API для онлайн-платформы, где пользователи могут создавать, проходить курсы и оплачивать обучение. Проект реализован с использованием Django и Django REST Framework.

## 📦 Основные возможности

- Регистрация и управление пользователями (авторизация по email)
- JWT-авторизация (получение и обновление токенов)
- Создание и редактирование курсов и уроков
- Загрузка аватарок, превью курсов и уроков
- Полный CRUD для курсов (через ViewSet) и уроков (через Generic-классы)
- Отображение списка уроков и их количества внутри курса
- Система платежей: оплата курсов или отдельных уроков
- Фильтрация и сортировка списка платежей
- **Валидация ссылок**: разрешены только видео с YouTube (`youtube.com`, `youtu.be`)
- **Подписка на курсы**: пользователь может подписаться/отписаться от обновлений курса
- **Пагинация**: все списки курсов и уроков разбиты на страницы (по 5 элементов)
- Гибкая система прав доступа:
  - **Обычные пользователи**: видят, редактируют и удаляют только свои курсы и уроки
  - **Модераторы** (группа `moderators`): могут просматривать и редактировать любые курсы и уроки, но **не могут создавать и удалять**
- Безопасность профиля:
  - Пользователь может редактировать только свой профиль
  - Просмотр чужого профиля доступен, но отображаются только публичные данные (`email`, `city`)
- Удобная админка для управления пользователями, группами, курсами, уроками, платежами и подписками
- **Покрытие тестами**: автоматические тесты для ключевых сценариев (CRUD, права, валидация, подписки)

## 🧱 Структура проекта

- **`users`** — приложение для управления пользователями и платежами  
  - Кастомная модель `User`: email (основной идентификатор), телефон, город, аватар  
  - Модель `Payment`: пользователь, дата оплаты, оплаченный курс **или** урок, сумма, способ оплаты (наличные / перевод)

- **`materials`** — приложение для учебных материалов  
  - Модель `Course`: название, описание, превью, владелец  
  - Модель `Lesson`: название, описание, превью, ссылка на видео, привязка к курсу, владелец  
  - Модель `Subscription`: связь «пользователь — курс» для управления подписками  
  - Валидатор `validate_youtube_url` — проверка ссылок на видео  
  - Пагинатор `MaterialsPagination` — управление количеством элементов на странице
